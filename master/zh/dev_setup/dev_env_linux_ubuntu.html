# Ubuntu LTS/Debian Linux 的开发环境

Ubuntu linux lts (opens new window)（16.04）是标准的/首选的 Linux 开发操作系统。 Linux允许您构建所有PX4目标（基于NuttX的硬件、高通骁龙飞控硬件、基于Linux的硬件、仿真、ROS）。

以下说明说明了如何 手动 设置每个受支持的目标的开发环境。

  • ubuntu.sh (opens new window): Installs Gazebo 9 and jMAVSim simulators and/or NuttX/Pixhawk tools. 不包含FastRTPS所依赖的工具。
  • ubuntu_sim.sh: ubuntu_sim_common_deps.sh + Gazebo8 模拟器。

提示

The scripts have been tested on clean Ubuntu 18.04 LTS and Ubuntu 20.04 LTS installations. They may not work as expected if installed "on top" of an existing system, or on a different Ubuntu release.

To build for Ubuntu 20.04 (focal) you must use docker (the GCC toolchain on Ubuntu 20.04 can build PX4, but the generated binary files are too new to run on actual Pi). For more information see PilotPi with Raspberry Pi OS Developer Quick Start > Alternative build method using docker.

    bash ubuntu.sh --no-nuttx --no-sim-tools

# 如何使用脚本

Ubuntu software repository provides a set of pre-compiled toolchains. Note that Ubuntu Focal comes up with gcc-9-arm-linux-gnueabihf as its default installation which is not fully supported, so we must manually install gcc-8-arm-linux-gnueabihf and set it as the default toolchain. This guide also applies to earlier Ubuntu releases (Bionic). The following instruction assumes you haven't installed any version of arm-linux-gnueabihf, and will set up the default executable with update-alternatives. Install them with the terminal command:

sudo usermod -a -G dialout $USER

Set them as default:

sudo apt-get remove modemmanager

# FastRTPS 安装

eProsima Fast RTPS (opens new window) is a C++ implementation of the RTPS (Real Time Publish Subscribe) protocol. FastRTPS is used, via the RTPS/ROS2 Interface: PX4-FastRTPS Bridge, to allow PX4 uORB topics to be shared with offboard components.

Follow the instructions in Fast RTPS Installation to install it.
